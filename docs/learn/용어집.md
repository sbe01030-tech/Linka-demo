# 용어집 — 심플 사전

> 백엔드·동료·AI가 쓰는 단어를 들었을 때 뜻만 알면 됩니다. 외울 필요 없고, 막힐 때마다 여기 들러서 한 줄씩 보세요.

---

## 1. 앱 / 코드 기본

| 용어 | 한 줄 |
|---|---|
| **컴포넌트 (Component)** | 화면을 이루는 부품 한 덩어리. 버튼, 카드, 입력창 같은 것 |
| **Props** | 부모 컴포넌트가 자식에게 내려주는 값. "이 버튼은 빨간색이야" 같은 |
| **State** | 컴포넌트가 자기 안에서 기억하는 값. "지금 입력창에 뭐가 써있나" |
| **Hook** | `useState`, `useEffect` 같은 `use~`로 시작하는 함수. 상태·효과 다루는 도구 |
| **JSX / TSX** | HTML처럼 보이는 React 문법. `.tsx`는 TypeScript 버전 |
| **Render** | 화면에 그리는 행위. "리렌더링됐다" = "다시 그려졌다" |
| **Mock 데이터** | 백엔드 없이 흉내낸 가짜 데이터. 지금 우리 앱이 이 상태 |

## 2. 네트워크 / API

| 용어 | 한 줄 |
|---|---|
| **API** | 서버와 대화하는 약속. "헬퍼 목록 줘" 같은 요청·응답 규칙 |
| **REST** | API 설계 스타일. URL + 메서드(GET/POST 등) 조합으로 표현 |
| **Endpoint** | API의 한 주소. `/api/v1/partners` 같은 거 한 줄 |
| **JSON** | 데이터 주고받는 형식. `{ "name": "Sari", "rating": 4.9 }` |
| **GET** | "달라" 요청. 검색·조회 |
| **POST** | "만들어달라" 요청. 회원가입·예약 생성 |
| **PUT / PATCH** | "수정해달라". PUT은 통째로, PATCH는 일부만 |
| **DELETE** | "지워달라" |
| **Header** | 요청에 끼워보내는 부가정보. 인증 토큰·언어 같은 거 여기 들어감 |
| **Body** | 요청·응답의 본문. JSON이 여기 담김 |
| **Payload** | Body에 담긴 실제 데이터. "이 회원가입 payload 좀 봐줘" |
| **Query string** | URL 뒤에 붙는 `?keyword=sari&limit=20` 같은 부분 |
| **Status code** | 응답 결과 숫자. 200=성공, 4xx=내 잘못, 5xx=서버 잘못 |

## 3. 자주 보는 에러 / 상태코드

| 코드 / 에러 | 한 줄 의미 | 누구 책임? |
|---|---|---|
| **200** | 성공 | — |
| **201** | 만들어졌어요 (POST 응답) | — |
| **400** | 요청이 잘못됨 (필드 빠짐 등) | **프론트** |
| **401 Unauthorized** | 로그인 안 됨 / 토큰 만료 | **프론트** (토큰 다시 보내야) |
| **403 Forbidden** | 권한 없음 | 정책 확인 |
| **404 Not Found** | 그런 거 없음 | URL 오타 또는 데이터 없음 |
| **429 Too Many Requests** | 너무 자주 호출함 (rate limit) | **프론트** (재시도 간격 조정) |
| **500 Internal Server Error** | 서버 폭발 | **백엔드** |
| **502 / 503 / 504** | 서버 다운·타임아웃 | **백엔드/인프라** |
| **CORS error** | 브라우저 보안. 서버가 우리 출처 허용 안 함 | **백엔드** (모바일 앱은 안 겪음) |
| **Network request failed** | 네트워크 자체가 끊김 / URL 잘못 | 둘 다 가능, 우선 URL 확인 |
| **undefined is not an object** | 없는 값에서 뭘 꺼냄 (e.g. `partner.name` 인데 partner가 undefined) | **프론트** (옵셔널 체이닝 `?.` 사용) |

## 4. 인증 / 보안

| 용어 | 한 줄 |
|---|---|
| **JWT (제이덜뜨)** | 로그인 후 받는 토큰 한 덩어리. 이후 요청마다 헤더에 끼워보냄 |
| **Access Token** | 짧은 유효기간 (보통 15분~1시간). 매 요청에 사용 |
| **Refresh Token** | 긴 유효기간. Access Token 만료되면 이걸로 새로 받음 |
| **OAuth** | 카카오·구글 로그인 같은 외부 인증 표준 |
| **HTTPS / SSL** | 통신 암호화. 운영 서버는 무조건 HTTPS |

## 6. Git

| 용어 | 한 줄 |
|---|---|
| **Repository (repo)** | 코드 저장소 한 단위 |
| **Commit** | 작업 완료 시점 박제 한 번. 메시지 남김 |
| **Branch** | 작업 분기. `main`은 본진, `feature/xxx`는 별도 작업선 |
| **Push** | 내 컴퓨터 → GitHub 서버로 업로드 |
| **Pull** | GitHub 서버 → 내 컴퓨터로 내려받기 |
| **Merge** | 두 브랜치 합치기 |
| **PR (Pull Request)** | "내 브랜치를 main에 합쳐주세요" 라는 합치기 요청 + 리뷰 창구 |
| **Conflict** | 같은 줄을 두 사람이 다르게 바꿔서 자동 합치기 실패. 손으로 해결 |
| **Rebase** | 브랜치 기준점을 다시 잡는 고급 작업. 처음엔 피해도 됨 |
| **Stash** | 작업 임시 보관 (다른 일 잠깐 해야 할 때) |

## 9. 백엔드 회의에서 자주 듣는 표현

| 표현 | 한 줄 의미 |
|---|---|
| "스펙 맞추자" | API 요청·응답 형식 합의하자 |
| "엔드포인트 정의" | URL과 메서드 정하자 (예: `POST /login`) |
| "페이로드" | 주고받는 본문 데이터 |
| "필드 추가요" | 응답에 항목 하나 더 넣어달라 |
| "스키마" | DB 테이블 또는 응답 형식 구조 |
| "마이그레이션" | DB 구조 바꾸기 |
| "스테이징" | 운영 직전 테스트 환경 |
| "롤백" | 이전 버전으로 되돌리기 |
| "캐시" | 반복 호출 줄이려고 임시 저장한 응답 |
| "히트한다" | "캐시에 있다" 또는 "잘 동작한다" |
| "옵저버블" | 실시간으로 변하는 값 (보통 WebSocket 맥락) |
| "rate limit 걸어둘게요" | 너무 자주 호출하면 막아버린다는 뜻 |

## 10. 헷갈리기 쉬운 짝

| A | B | 차이 |
|---|---|---|
| **Frontend** | **Backend** | 사용자가 보는 쪽 / 서버 쪽 |
| **Client** | **Server** | 요청 보내는 쪽 / 응답 주는 쪽 |
| **Library** | **Framework** | 가져다 쓰는 도구 / 우리가 그 안에 들어가서 따라야 하는 큰 틀 |
| **GET vs POST** | — | 가져오기 / 만들기·바꾸기 |
| **Path vs Query** | — | `/users/123` (path) vs `/users?role=helper` (query) |
| **Sync vs Async** | — | 순서대로 / 비동기. 네트워크는 거의 다 async |
| **Auth vs Authz** | — | Authentication(누구냐) / Authorization(뭘 할 수 있냐) |

## 11. "이 단어 모르겠으면 일단 이렇게"

- **CORS 에러** → 백엔드에 "출처 허용 좀요"
- **401 떴는데** → 토큰이 만료됐거나 안 보낸 거. 로그인 흐름 확인
- **500 떴는데** → 백엔드 책임. 로그 같이 봐달라고 요청
- **타임아웃** → 서버 응답이 너무 늦음. 백엔드에 알리기
- **undefined is not an object** → 코드에서 빈 값 쓰려는 곳 찾기. `?.` 추가
- **Network request failed** → URL 오타거나 서버 다운. URL부터 확인
