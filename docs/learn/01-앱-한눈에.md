# 01. Linka 앱 한눈에 (폴더 구조)

> 목표: 어떤 파일이 어디 있고, 무엇을 하는지 머릿속 지도를 그립니다.
> 시간: 약 15분.

---

## 큰 그림

Linka는 **React Native(Expo)** 로 만든 모바일 앱입니다. 한 줄로 표현하면:

> "JavaScript/TypeScript로 코드를 짜면, iOS와 Android 앱 둘 다 동시에 만들어주는 도구."

직접 iOS 코드(Swift), Android 코드(Kotlin)를 짤 필요가 없다는 게 핵심입니다.

---

## 최상위 폴더

프로젝트 루트(`/Users/bigband/Linka-app/Linka-app`)에 있는 폴더들:

```
Linka-app/
├── src/              ← 우리가 매일 만지는 코드. 90%는 여기서 작업합니다.
├── assets/           ← 이미지·아이콘·폰트 같은 정적 파일
├── android/          ← Android 빌드용 (대부분 자동 생성, 거의 안 만짐)
├── ios/              ← iOS 빌드용 (대부분 자동 생성, 거의 안 만짐)
├── node_modules/     ← 설치된 라이브러리들 (자동 생성, 절대 직접 안 만짐)
├── docs/             ← 우리가 만든 기획·학습 문서
├── App.tsx           ← 앱의 진입점 (가장 처음 실행되는 파일)
├── package.json      ← 어떤 라이브러리를 쓰는지 적힌 명세서
├── package-lock.json ← 정확한 라이브러리 버전 잠금 (자동)
├── tsconfig.json     ← TypeScript 설정
└── app.json          ← Expo 설정 (앱 이름·아이콘 등)
```

**기억할 것 딱 하나**: `src/` 가 본진. 나머지는 대부분 자동 생성이라 평소엔 신경 안 써도 됩니다.

---

## src/ 안

```
src/
├── screens/      ← 화면(페이지). 가장 많이 작업하는 곳
├── components/   ← 여러 화면이 공유하는 작은 부품들
├── navigation/   ← 화면 간 이동 규칙 (탭·스택)
├── store/        ← 앱 전역 상태 (로그인 정보·언어 같은 것)
├── i18n/         ← 다국어 번역 텍스트
├── constants/    ← 색상·이미지 경로·고정값
└── types/        ← TypeScript 타입 정의
```

### 화면(screens) — 폴더 구조

```
src/screens/
├── auth/        ← 로그인·가입·환영
├── customer/    ← 고객 화면 (홈·검색·예약·프로필)
├── worker/      ← 헬퍼 화면 (헬퍼용 홈·주문·프로필)
├── driver/      ← 드라이버 화면
├── errand/      ← 심부름 화면
├── chat/        ← 채팅 목록·채팅방
├── community/   ← 커뮤니티 (제거 예정)
├── map/         ← 지도
├── profile/     ← 프로필 수정
└── help/        ← 도움말 / FAQ
```

각 폴더 안엔 `XxxScreen.tsx` 파일들이 있어요. 예: `MapScreen.tsx`, `LoginScreen.tsx`.

---

## 공식: "이 파일이 뭐 하는 파일이지?"

파일명만 봐도 짐작할 수 있는 패턴이 있어요:

| 파일명 | 뜻 |
|---|---|
| `XxxScreen.tsx`     | 한 화면(페이지) 전체 |
| `XxxCard.tsx`       | 카드 모양 부품 |
| `XxxModal.tsx`      | 팝업 창 |
| `XxxButton.tsx`     | 버튼 부품 |
| `useXxx.ts`         | 재사용 가능한 로직(훅) |
| `xxxStore.ts`       | 상태 저장소 |
| `index.tsx` / `index.ts` | 폴더의 대표 파일 |

이 패턴만 알아도 모르는 파일을 봤을 때 "아, 화면이구나" / "아, 작은 부품이구나" 짐작이 됩니다.

---

## 실전 — 이 화면은 어디서 시작될까?

**시나리오:** 환영 화면(가입 카드 3개 있는 그 화면)의 텍스트를 바꾸고 싶다.

찾는 순서:

1. 화면 이름이 "환영" → 영어로 "Welcome"
2. `src/screens/auth/` 폴더로 이동 (로그인·가입 계열은 auth)
3. `WelcomeScreen.tsx` 발견
4. 이 파일 안에 텍스트가 있음

→ 실제 파일: [src/screens/auth/WelcomeScreen.tsx](../../src/screens/auth/WelcomeScreen.tsx)

이 검색 흐름이 자동으로 떠오르면 1단계 끝.

## App.tsx — 모든 것의 시작

[App.tsx](../../App.tsx) 가 앱이 켜졌을 때 가장 먼저 실행되는 파일입니다. 여기서:

1. 폰트 같은 초기 설정을 불러오고
2. 네비게이션 시스템을 시작하고
3. 첫 화면(보통 SplashScreen)을 띄웁니다.

평소엔 거의 안 만져도 되지만, "앱 켜자마자 뭐가 동작하지?" 의문이 들면 여기를 봅니다.
